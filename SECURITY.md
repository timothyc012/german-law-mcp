# Security Policy

## Supported Versions

Security fixes are applied to the latest maintained version of this project.
If you are unsure whether an issue affects a supported version, report it anyway.

## Reporting a Vulnerability

Please do not open public GitHub issues for suspected security vulnerabilities.

Instead, report privately using one of these channels:
- GitHub Security Advisories / private vulnerability reporting, if enabled for the repository
- Direct maintainer contact for this project owner (`timothyc012`) with the subject line `SECURITY: german-law-mcp`

Please include:
- a clear description of the issue
- affected version / commit if known
- reproduction steps or proof of concept
- potential impact
- any suggested mitigation

## Response Expectations

Target initial response time: within 72 hours.

After confirmation:
- we will assess severity and impact
- we may ask for reproduction details
- we will aim to coordinate a fix before public disclosure

## Disclosure Guidance

Please avoid public disclosure until a fix or mitigation is available.
If a report was accidentally posted publicly, maintainers may move the discussion to a private channel and redact sensitive details where possible.

## Scope

Report issues related to:
- authentication or authorization bypass
- unsafe tool behavior or sensitive data disclosure
- dependency or supply-chain concerns
- transport, scraping, or parser vulnerabilities with security impact
- privacy-impacting flaws

Non-security bug reports should go through the normal issue tracker.
